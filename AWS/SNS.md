## 機能
- A2Aメッセージング
	- メッセージング内容をS3や、Redshiftなどの分析テーブルに入れることも可
- A2Pメッセージング
- メッセージのフィルター処理
- 標準及びFIFOトピック
- メッセージ耐久性
	- 公開済みのメッセージは複数のサーバー、データセンター間で保存される
	- 配信の再試行ポリシーが設定できる
	- デッドレターキューで再試行ポリシー実行前のメッセージを保持
- メッセージ属性
- メッセージのフィルター処理
- メッセージセキュリティ(KMSの暗号化キーを用いた暗号化)

## 主な要素
- トピック
	- アプリケーションからの受取先
	- FIFOトピックとスタンダートトピックが選べる、順序付けが重要ならFIFO、必要ない場合はスタンダードを使う
	- そのほか暗号化、アクセスポリシー、アーカイブポリシー等で受け取り時の詳細設定ができる
- サブスク
	- メッセージを受け取るエンドポイントの設定
	- 以下のプロトコルを設定できる
		- SQS
		- Lambda
		- HTTP
		- HTTPS
		- Firehose
		- SMS
		- eメール
		- Json形式のメール
		- プラットフォームアプリケーションのエンドポイント
	- 受け取るメッセージのフィルターと、配信失敗時に入れるキューの設定が出来る
